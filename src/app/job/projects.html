As a software development intern, I am a member of Development Team of IT Department. My main task is helping maintain the database and ftp to support our clients which include more than 270 senior livings, 10,000 staffs and over 100,000 customers.

Maintain FTP and Database

Customers, employees, and so many special request or case can generate a report. More than 500 reports are reported every day. Among these reports, over 30 categories, 10 access levels, and 20 organizations exist. Transferring these reports to the right person can be difficult especially if all of these information are bad structured in database.

Here is what I do. First of all, instead of separating single information to different tables and using a lot of "joins" in a sql query, I integrated "FTP" related information to the "report" table, which significantly simplify sql queries. Then, I modularized stored procedures to make each procedure do only one single task. By doing this, instead of writing new procedures when meeting a new task, we can just combine them.

Previously, we use a server to store all of the reports. And the directory is bad structured. So I helped transfer reports to a file system, and re-built structures.

Convert VB to T-SQL

They used to use VB to achieve all of FTP operations. This can be easier since the object oriented language make relationships clear. But the problem is, it's hard to run the code everywhere. After all, Visual Studio and the Runtime Environment is too "heavy-weight"

To make it easier to manipulate, we decided to transfer all of those VB code to T-SQL, which can be easily operated in a SQL Server Client. This may sound a easy job, but actually it's like nightmare for me. First of all, I only have the "READ" privilege, which means it's impossible for me to test my code. Secondly, not fully understanding the whole structure of FTP related table's structure and the bossiness logic behind them, in some cases I had to guess what the original VB code want to do. What's more, since I was only given some part of the VB code, when they reference another object, how can I know what properties and methods they have?

Even though I really met some difficultly, with the help of others and my hardworking, finally I worked them out.

Applied WinSCP as new FTP client

Previously we used Microsoft's FTP client, but for some reason that I don't know, they want to use a new FTP Client ——WinSCP.

To achieve this, I used WInSCP's syntax to replace previous ftp syntax in those t-sql files. This is not only a word by word translation. Sometimes I need to change the original t-sql structure. So a fully understanding of the files is needed.

Solely deployed front-end

Since I already made this website RESTful, it's better to separate front-end and back-end. By doing this, you can either develop front-end or back-end without causing unexpected bugs.

For the static application which doesn't need start up time like this, Heroku provide free deploy service. To make it compiled to runtime on Heroku, we need give this application a "php like idnex page" with a tag <?php include_once("home.html"); ?>. This tag will redirect to our html homepage.

Different Layers

First layer is RESTful API Layer. This layer accept request or data from http. To operate received before send to database, or format data thar retrieved from database, API layer will communicate and only communicate with Logic Layer.

Logic Layer will invoke a third-party service such as the Cloudnary Remote File System, Gravatar or Gson. This layer is the converter that convert the data object that the DAO can configure, and the data format the API can read. Don't worry about the huge amount of objects, I already manually "null" those unnecessary but still reachable objects. still

DAO, AKA Database Access Object, is the object we can use to operate our database. I use inheritance and combination to build structure of DAOs. I also use Hibernate as the DB Mapper. Since connections have timeout, and not much people visit my website and use connections, always visitors need to wait a few seconds to start up the whole Hibernate Framework.

Design Pattern is a good thing. It can help me manage and extend code easier. I use a lot of design patterns such as singleton, builder and so on.

Remote Database and File System

I use a remote Postgresql provides by Amazon Web Service. By doing so, my data is accessable on every time.

No one wants to store files on his server. So do I. By using Cloudnary's remote File System, I can easily manage photos or files without changing front-end code and re-deploying it.

Testing and Environments

Code without testing is not code. Directed by TDD, I write unite test for all methods. For those methods that reference other objects, Mockito is introduced here as a mock testing tool. Not only this, but also I provided integration testing to make sure every function chain works.
